# 미네르바 타이머

미네르바 타이머는 방송용 후원 목표 타이머/카운터 데스크톱 앱입니다.

앱에서 목표를 설정하고, OBS에는 로컬 오버레이 주소를 Browser Source로 추가해서 사용합니다. 별도의 웹 대시보드가 아니라 내 컴퓨터에서 실행되는 앱입니다.

## 다운로드

최신 버전은 아래 릴리즈 페이지에서 받을 수 있습니다.

[Minerva Timer v0.1.0 다운로드](https://github.com/yomangneko/minerva_timer-public/releases/tag/v0.1.0)

운영체제에 맞는 파일을 받으세요.

| 운영체제 | 받을 파일 |
| --- | --- |
| Windows | `minerva-timer-...-win-x64-....exe` |
| macOS Apple Silicon | `minerva-timer-...-mac-arm64-....dmg` |
| macOS 압축본 | `minerva-timer-...-mac-arm64-....zip` |

## 설치

### Windows

1. `.exe` 파일을 다운로드합니다.
2. 실행해서 설치합니다.
3. Windows SmartScreen 경고가 나오면 게시자 미확인 앱으로 표시될 수 있습니다.

### macOS

1. `.dmg` 또는 `.zip` 파일을 다운로드합니다.
2. 앱을 Applications 폴더로 옮기거나 압축을 푼 뒤 실행합니다.
3. macOS 보안 경고가 나오면 `시스템 설정 > 개인정보 보호 및 보안`에서 실행을 허용해야 할 수 있습니다.

현재 빌드는 개발자 서명/공증이 적용되지 않은 배포본입니다.

## 기본 사용법

1. 미네르바 타이머 앱을 실행합니다.
2. 타이머 또는 카운터 목표를 설정합니다.
3. 앱에 표시되는 OBS 오버레이 URL을 복사합니다.
4. OBS에서 `Browser Source`를 추가합니다.
5. URL에 복사한 오버레이 주소를 붙여 넣습니다.
6. 앱에서 타이머/카운터를 조작하면 OBS 오버레이가 바로 갱신됩니다.

기본 오버레이 주소는 로컬 주소입니다.

```text
http://127.0.0.1:17770/overlay
```

포트가 이미 사용 중이면 앱이 다른 로컬 포트를 사용할 수 있습니다. 이 경우 앱 화면에 표시되는 주소를 OBS에 넣으면 됩니다.

## 주요 기능

- 타이머 시작, 일시정지, 종료, 초기화
- 시간 추가/차감
- 카운터 목표값과 진행값 관리
- 후원 금액에 따른 타이머/카운터 반영 설정
- CHZZK/CIME 연결 카드
- 테스트용 모의 후원 시뮬레이터
- OBS용 로컬 오버레이
- 앱 재시작 후 설정 유지

## 방송에서 쓰는 방법

OBS에서 Browser Source를 만들 때 아래처럼 설정하는 것을 권장합니다.

| 항목 | 권장값 |
| --- | --- |
| URL | 앱에 표시된 오버레이 URL |
| 너비 | 800 |
| 높이 | 400 |
| 사용자 지정 CSS | 필요할 때만 사용 |

오버레이는 인터넷 공개 주소가 아니라 내 컴퓨터의 로컬 주소로 동작합니다.

## 연결이 안 될 때

### OBS 오버레이가 안 보임

- 미네르바 타이머 앱이 켜져 있는지 확인하세요.
- OBS에 입력한 URL이 앱에 표시된 URL과 같은지 확인하세요.
- `127.0.0.1` 주소는 같은 컴퓨터에서만 열립니다.
- 앱을 재시작한 뒤 표시된 URL을 다시 복사해 보세요.

### 후원 연결이 안 됨

- 앱의 연결 카드에 표시되는 상태 메시지를 확인하세요.
- 브라우저 로그인 후 앱으로 돌아와 상태가 바뀌는지 확인하세요.
- 권한 부족 메시지가 나오면 해당 플랫폼의 후원 조회 권한을 확인해야 합니다.

### 설정이 꼬였을 때

- 앱 안의 초기화/재설정 버튼을 먼저 사용하세요.
- 같은 플랫폼 계정 연결이 반복 실패하면 연결 초기화 후 다시 연결하세요.

## 보안 안내

- 이 저장소는 사용자 다운로드용 public 저장소입니다.
- 서버 주소, provider client secret, DB URL, 암호화 키는 README나 릴리즈 설명에 공개하지 않습니다.
- 데스크톱 앱에는 provider client secret을 넣지 않는 구조입니다.
- OBS 오버레이는 기본적으로 로컬 컴퓨터에서만 열리는 주소를 사용합니다.

## 개발자용 소스 코드

이 public 저장소에는 전체 개발 소스 코드를 올리지 않습니다.

사용자는 릴리즈 페이지에서 설치 파일만 받으면 됩니다. 개발/운영 소스는 별도 비공개 저장소에서 관리하는 것을 전제로 합니다.
